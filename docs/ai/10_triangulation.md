## 1. 位置づけ

[9. カメラ姿勢の推定](./9_camera_pose_estimation.md)では、3D 点の位置が既知でカメラの姿勢が未知だった。この文書はそのちょうど**逆問題**を扱う。姿勢が既知の 2 台以上のカメラに同じ点が写っているとき、その点の **3D 座標**を復元する。これが**三角測量** (triangulation) である。

原理は測量そのものである。各カメラについて「カメラ中心から、その点が写ったピクセルの方向へ伸びる半直線 (**視線**)」が引ける。点はどのカメラの視線の上にも乗っているはずだから、**視線の交点**が点の位置である。2 台のカメラがあれば視線は 2 本になり、交点が 1 点に定まる — これが「三角」測量と呼ばれる所以で、2 つの視点と対象点が作る三角形を解いていることになる。

面白いのは、この問題が 9 番と違って**線形**に解けることである。LM 法は主役ではなく、仕上げ磨きとして脇役で登場する。使う道具は [1. 線形最小二乗法](./1_least_squares_method.md)と [2. QR 分解](./2_qr_decomposition.md)であり、初期の文書で作った部品がここで実戦投入される。

- 第 3 節: **線形三角測量** — 視線の条件を線形方程式に直し、QR 分解による最小二乗で解く
- 第 4 節: **中点法** — 「各視線に一番近い点」を求める幾何的な解法
- 第 5 節: **非線形リファイン** — 再投影誤差を LM 法で磨く (9 番の部品の再利用)
- 第 6 節: **退化** — 視差が小さいと奥行きが決まらなくなる現象

サンプルコードは `src/bin/10_triangulation.rs` (線形三角測量は 11 番でも使うため `src/geometry.rs` に置いた)。

## 2. 問題設定 — 視線と正規化画像座標

記号は 9 番と同じである。カメラ $i$ の姿勢 $(R_i, \mathbf{t}_i)$ と内部パラメータ ($f, c_x, c_y$) は既知、未知なのはワールド座標の点 $\mathbf{p} \in \mathbb{R}^3$ だけである。点はカメラ座標 $\mathbf{P}_c = R_i \mathbf{p} + \mathbf{t}_i$ に写り、スクリーン座標は $\pi(\mathbf{P}_c) = [f X / Z + c_x, \; f Y / Z + c_y]^\top$ だった。

観測 $\mathbf{u}_i = [u_i, v_i]^\top$ からは、投影の式を逆に解くことで「その点がカメラから見て**どの方向**にあるか」が分かる。

```math
\hat{x}_i = \frac{u_i - c_x}{f}, \qquad \hat{y}_i = \frac{v_i - c_y}{f}
```

この $(\hat{x}_i, \hat{y}_i)$ を**正規化画像座標**と呼ぶ。投影の式と見比べると $\hat{x}_i = X / Z$、$\hat{y}_i = Y / Z$ なので、点はカメラ座標系で方向ベクトル

```math
\mathbf{d}_i = [\hat{x}_i, \; \hat{y}_i, \; 1]^\top
```

の**スカラー倍**の位置にある (奥行き $Z$ を掛ければ $\mathbf{P}_c = Z \mathbf{d}_i$ に戻る)。つまり 1 つの観測は「点はこの方向の半直線上にある」という情報であり、方向は分かるが**奥行きは分からない**。奥行きを確定させるのが 2 台目のカメラの役割である。

なお、ノイズがある現実の観測では 2 本の視線は厳密には交わらない (3 次元空間の 2 直線は一般に**ねじれの位置**にある)。だから「交点を求める」は正確には「すべての視線に一番近い点を求める」という**最小二乗問題**になる。ここが 1 番以来のテーマとつながるところである。

## 3. 線形三角測量 (DLT)

### 3.1 視線の条件を線形方程式にする

カメラ $i$ で、点のカメラ座標 $\mathbf{P}_c = R_i \mathbf{p} + \mathbf{t}_i$ が方向 $\mathbf{d}_i = [\hat{x}_i, \hat{y}_i, 1]^\top$ と平行である、という条件を式にする。成分の比が揃えばよいので

```math
\hat{x}_i \, ( \mathbf{r}_3^\top \mathbf{p} + t_3 ) = \mathbf{r}_1^\top \mathbf{p} + t_1,
\qquad
\hat{y}_i \, ( \mathbf{r}_3^\top \mathbf{p} + t_3 ) = \mathbf{r}_2^\top \mathbf{p} + t_2
```

となる ($\mathbf{r}_k^\top$ は $R_i$ の第 $k$ 行、$t_k$ は $\mathbf{t}_i$ の第 $k$ 成分。1 本目は $X = \hat{x} Z$、2 本目は $Y = \hat{y} Z$ を書き下したもの)。ここで重要なのは、この 2 本が未知数 $\mathbf{p}$ について**線形**なことである。9 番では「$Z$ で割る」せいで非線形になったが、分母を払った形で条件を書けば割り算が消える。整理すると

```math
( \hat{x}_i \, \mathbf{r}_3^\top - \mathbf{r}_1^\top ) \, \mathbf{p} = t_1 - \hat{x}_i t_3,
\qquad
( \hat{y}_i \, \mathbf{r}_3^\top - \mathbf{r}_2^\top ) \, \mathbf{p} = t_2 - \hat{y}_i t_3
```

という、見慣れた $A \mathbf{p} = \mathbf{b}$ の 2 行分になる。

### 3.2 積んで最小二乗で解く

カメラが $m$ 台あれば式は $2m$ 本。未知数は 3 なので、$m \geq 2$ で方程式が足りる。$2m \times 3$ の行列 $A$ と右辺 $\mathbf{b}$ に積んで、線形最小二乗

```math
\min_{\mathbf{p}} \| A \mathbf{p} - \mathbf{b} \|^2
```

を解けばよい。サンプルコード (`geometry.rs` の `triangulate_linear`) は [2 番](./2_qr_decomposition.md)で作った `lstsq_qr` にそのまま渡している。この解き方は **DLT** (Direct Linear Transform) と呼ばれる。ノイズがなければ厳密解 (視線の交点そのもの) が、ノイズがあれば「式の残差を最小にする点」が得られる。

### 3.3 代数的誤差という注意点

DLT が最小化している残差は、実は再投影誤差そのものではない。1 本目の式の残差を変形すると

```math
\hat{x}_i ( \mathbf{r}_3^\top \mathbf{p} + t_3 ) - ( \mathbf{r}_1^\top \mathbf{p} + t_1 )
= \hat{x}_i Z - X
= Z \Bigl( \hat{x}_i - \frac{X}{Z} \Bigr)
```

つまり「正規化座標での再投影誤差 $\times$ **奥行き $Z$**」である。このような、幾何的な誤差に余計な係数が掛かった量を**代数的誤差** (algebraic error) と呼ぶ。奥行きの深い点ほど重く扱われるため、点ごとの奥行きが大きく違う場合、DLT の解は再投影誤差の意味では最適でない。とはいえ通常は十分良い近似であり、必要なら次の非線形リファイン (第 5 節) で仕上げればよい。「まず線形化した代理の問題を解いて初期値を得て、本当の目的関数は非線形最適化で磨く」という、[4 番](./4_nonlinear_least_squares.md)以来の定石の一例である。

なお教科書でよく見る DLT は、$\mathbf{p}$ を同次座標 (4 次元) にして $A \tilde{\mathbf{p}} = \mathbf{0}$ 型の同次方程式を立て、[3. 特異値分解](./3_singular_value_decomposition.md)の最小特異値方向として解く変種である。無限遠点 (視線が完全に平行な極限) も表現できる利点があるが、本文書の範囲では上の非同次形で十分なので、1〜2 番の道具で解ける形を選んだ。

## 4. 中点法 — 幾何で解く

同じ問題を、方程式ではなく幾何で考えることもできる。視線 $i$ は「カメラ中心 $\mathbf{C}_i = - R_i^\top \mathbf{t}_i$ を通り、単位方向 $\hat{\mathbf{d}}_i$ (カメラ座標の $\mathbf{d}_i$ をワールド座標に直して正規化したもの、$\hat{\mathbf{d}}_i \propto R_i^\top \mathbf{d}_i$) に伸びる直線」である。点 $\mathbf{p}$ からこの直線までの距離は、$\mathbf{p} - \mathbf{C}_i$ から「視線方向の成分」を取り除いた残りの長さ

```math
\mathrm{dist}_i(\mathbf{p}) = \bigl\| ( I - \hat{\mathbf{d}}_i \hat{\mathbf{d}}_i^\top ) ( \mathbf{p} - \mathbf{C}_i ) \bigr\|
```

で与えられる ($\hat{\mathbf{d}} \hat{\mathbf{d}}^\top$ は方向 $\hat{\mathbf{d}}$ への射影行列。ベクトルから射影成分を引く操作は [2 番](./2_qr_decomposition.md)の直交化と同じ発想である)。距離の 2 乗和 $\sum_i \mathrm{dist}_i(\mathbf{p})^2$ を最小化するため勾配を 0 と置くと ($P_i = I - \hat{\mathbf{d}}_i \hat{\mathbf{d}}_i^\top$ が対称かつ $P_i^2 = P_i$ であることを使う)、

```math
\Bigl( \sum_i P_i \Bigr) \mathbf{p} = \sum_i P_i \, \mathbf{C}_i
```

という $3 \times 3$ の連立 1 次方程式になり、`solve_linear` で解ける (サンプルコードの `triangulate_midpoint`)。カメラが 2 台のとき、この解は「2 本の視線の最接近点どうしを結ぶ線分の中点」になることが知られており、**中点法** (midpoint method) と呼ばれる。

中点法は「3D 空間での距離」を、DLT は「(奥行き重み付きの) 画像上のずれ」を最小化しており、ノイズがあると両者は微妙に違う答えを返す。どちらも係数行列が特異になるのは**すべての視線が平行**なときだけである (平行な方向が全 $P_i$ に共通の零方向になる)。それが実質的に起きかけるのが第 6 節の退化である。

## 5. 非線形リファイン — 本当の再投影誤差を磨く

最後に、本当に最小化したい量である再投影誤差

```math
E(\mathbf{p}) = \sum_{i} \| \pi ( R_i \mathbf{p} + \mathbf{t}_i ) - \mathbf{u}_i \|^2
```

を LM 法で磨く。未知数は $\mathbf{p}$ の 3 つだけで、9 番と比べてむしろ簡単である。ヤコビ行列は連鎖律で

```math
\frac{\partial \mathbf{r}_i}{\partial \mathbf{p}}
= \frac{\partial \pi}{\partial \mathbf{P}_c} \cdot \frac{\partial \mathbf{P}_c}{\partial \mathbf{p}}
= \frac{\partial \pi}{\partial \mathbf{P}_c} \cdot R_i
\qquad (2 \times 3)
```

となる。部品 1 (投影の微分 $2 \times 3$) は 9 番と同一で、新しい部品は $\partial \mathbf{P}_c / \partial \mathbf{p} = R_i$ ($\mathbf{P}_c = R_i \mathbf{p} + \mathbf{t}_i$ は $\mathbf{p}$ について線形なので微分は係数行列そのもの) だけである。9 番の 4 節で予告した「部品を 1 個足すだけ」がこれである。LM ループは 9 番と同じ簡素版 (Marquardt 変種 + 成功で 1/10、失敗で 10 倍) をそのまま使う。初期値には DLT の解を渡す。

## 6. 退化 — 視差が小さいと奥行きが決まらない

三角測量の精度は、2 本の視線が**どれだけ角度をなして交わるか** (**視差**、parallax) で決まる。カメラ間の距離 (**基線長**、baseline) が長いほど視差は大きい。

基線長を 0 に近づける極限を考えると分かりやすい。2 台のカメラが同じ場所にあれば視線は 2 本とも同じ半直線になり、交点は半直線全体、つまり**奥行きが完全に不定**になる。基線長が短いだけなら視線はわずかな角度で交わるが、浅い角度で交わる 2 直線の交点は、直線が少し揺れただけで交線方向に大きく滑る。観測ノイズによる視線方向の揺らぎは一定なので、**3D 誤差 (特に奥行き方向) は視差にほぼ反比例して増える**。

これは行列の言葉では $A$ の**条件数の悪化**である ([2 番](./2_qr_decomposition.md)・[3 番](./3_singular_value_decomposition.md)で学んだ、入力の小さな誤差が解で増幅される現象)。視線が平行に近づくと $A$ の 3 つの列が張る空間の「奥行き方向の感度」を担う特異値が 0 に近づき、その方向の誤差増幅 $1 / \sigma$ が爆発する。QR や SVD で丁寧に解いても救えない、**問題そのものの条件**の悪さである (解法の工夫で救えるのは「解き方による悪化」だけ、というのが 2〜3 番の教訓だった)。

実務では、視差が閾値以下の点を三角測量の対象から外す、復元した点の奥行きが負 (カメラの背後) になったら棄却する、といった防御が入る。

## 7. 実験

サンプルコード `src/bin/10_triangulation.rs` では、2 台のカメラ (1 台目は原点、2 台目は横に基線長だけ離れてシーン中心を向く。焦点距離 800) と 100 個の 3D 点 ($z = 4$〜$6$) で 3 つの実験を行う。実行は `cargo run --bin 10_triangulation`。

**実験 1: ノイズなし。** 線形三角測量の 3D 誤差の最大値は $5.7 \times 10^{-15}$、中点法は $2.0 \times 10^{-13}$。どちらも数値精度の限界まで厳密に復元する。視線が本当に交わるなら、交点を求めるだけの問題だからである。

**実験 2: ノイズ ±0.5 px (基線長 1.0)。** 3 手法の平均誤差は次のとおり。

| 手法 | 平均 3D 誤差 | 平均再投影誤差 $E$ |
| --- | --- | --- |
| 線形 (DLT) | 0.012599 | 0.078960 |
| 中点法 | 0.012602 | 0.078975 |
| 線形 + LM リファイン | 0.012603 | **0.078878** |

3 手法がほぼ横並びで、リファインの効果は再投影誤差の 4 桁目に現れる程度である。これは今回の配置が良条件 (十分な視差、奥行きのばらつきが小さい) で、DLT の「奥行き重み」の歪みがほとんど効かないためである。逆に言うと、**良条件の三角測量では線形解がすでにほぼ最適**であり、リファインが本領を発揮するのは奥行きの差が大きいシーンや視点数が多い場合である。なおリファインが 3D 誤差では最小になっていない (4 桁目でわずかに劣る) のは異常ではない。リファインが最小化するのは再投影誤差であって 3D 誤差ではなく、ノイズがある以上この 2 つの最小点は厳密には一致しないからである。

**実験 3: 基線長と精度。** ノイズ ±0.5 px のまま基線長を狭めていくと、平均 3D 誤差は次のように変わる。

| 基線長 | 2.0 | 1.0 | 0.5 | 0.2 | 0.1 | 0.05 |
| --- | --- | --- | --- | --- | --- | --- |
| 平均 3D 誤差 | 0.0071 | 0.0126 | 0.0243 | 0.0601 | 0.1207 | 0.2447 |

基線長を半分にするごとに誤差がほぼ 2 倍になっており、第 6 節の「視差に反比例」がそのまま数字に現れている。基線長 0.05 (視差約 0.6°) では、観測ノイズはわずか ±0.5 px なのに 3D 誤差は 0.24 (シーンの奥行き幅の 1 割超) に達する。

このほかテストコードでは、3 視点にすると 2 視点より精度が上がること、リファインが再投影誤差を悪化させないことも検証している。

## 8. まとめ

この文書の内容を一文でまとめる。

> 三角測量とは、姿勢既知のカメラ 2 台以上の観測から「点は各カメラの視線上にある」という条件を連立させて 3D 点を復元する問題であり、条件を分母を払って線形化すれば QR 分解による最小二乗で直接解け (DLT)、必要なら再投影誤差を LM 法で磨ける。精度は視差で決まり、基線長が短いと奥行きが急激に不確かになる。

これで「点が既知ならカメラが求まる」(9 番) と「カメラが既知なら点が求まる」(本文書) の両方向が揃った。しかし現実の Structure from Motion では、最初はカメラも点も**どちらも未知**である。鶏と卵のようなこの状況で両方を一度に推定するのが、次の[バンドル調整](./11_bundle_adjustment.md)である。
